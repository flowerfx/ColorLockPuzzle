SAParam object

A SAParam object represents a command bind variable or stored procedure parameter.

A command (an SQL statement or procedure) can have input and/or output parameters. Each parameter is represented by SAParam object. A set of SAParam objects is usually created implicitly by the Library or can be created explicitly by the user.

With the methods of SAParam object you can do the following actions:

  • Return the name of a parameter using Name method;
  • Set and return the direction type of a parameter (input, output, etc.) using setParamDirType and  ParamDirType method;
  • Set and return parameter's data type with the setParamType and ParamType methods;
  • Set and return parameter's data size with the setParamSize and ParamSize methods;

SAParam object methods provide the following possibilities to work with parameter's value:

  • Set the value of an input parameter with methods defined in the base class SAValue (setAsLong, setAsFloat, setAsDouble, setAsString and so on);
  • View the value of an output parameter with methods defined in the base class SAValueRead (asLong, asFloat, asDouble, asString and so on);
  • Organize piecewise reading of Long or BLob(CLob) value using ReadLongOrLob method.

Normally you should not create SAParam object yourself. The Library always does it implicitly when needed. To get SAParam object use SACommand::Param method. But if you have some reasons to create parameters explicitly call SACommand::CreateParam method. All parameters can be destroyed by calling SACommand::DestroyParams method.
